1. Підготовка до розгортання
a. Переконайтеся, що ваш проект готовий до продакшн-середовища
Оновіть файл .env:
Створіть або оновіть файл .env із продакшн-змінними (наприклад, MONGODB_URI, JWT_SECRET, PORT, NODE_ENV=production).
Приклад:

ini
Copy
PORT=3000
MONGODB_URI=your_production_mongodb_connection_string
JWT_SECRET=your_long_random_secret_key
NODE_ENV=production
Переконайтеся, що .env додано до .gitignore:
Щоб секретні дані не потрапили до репозиторію.

Переконайтеся, що всі залежності прописані у package.json.

b. Локальне тестування
Переконайтеся, що ваш додаток працює локально у режимі продакшн (наприклад, змініть NODE_ENV на production і протестуйте).

2. Розгортання на хостингу ukraine.com.ua
a. Доступ до хостингу та панелі керування
Увійдіть у ваш особистий кабінет на ukraine.com.ua.
Переконайтеся, що ваш тарифний план підтримує Node.js.
– Деякі хостинги пропонують окрему підтримку Node.js через cPanel або спеціальну панель керування.
b. Завантаження файлів
FTP / File Manager:
– Використовуйте FTP-клієнт (наприклад, FileZilla) або вбудований файловий менеджер у панелі керування, щоб завантажити всі файли вашого проекту (включаючи package.json, .env, ваш код, папки з ресурсами тощо) на сервер.
– Розмістіть їх у потрібній теці, наприклад, в папці public_html/node_app або в іншій, що підтримує Node.js.

Встановіть залежності:
– Якщо у вашій панелі керування є можливість виконання команд (наприклад, через SSH або через Web Terminal), перейдіть у директорію з вашим проектом і виконайте:

bash
Copy
npm install
– Якщо немає доступу до SSH, перевірте, чи на хостингу є спеціальна опція для встановлення залежностей.

c. Налаштування середовища та запуск додатку
Налаштування змінних оточення:
– За допомогою панелі керування (якщо підтримується) або завантажте файл .env (переконайтеся, що він містить продакшн-змінні).

Запуск Node.js додатку:
– Деякі хостинги дозволяють налаштувати Node.js-додаток через панель (наприклад, задавши шлях до файлу запуску, порт, тощо).
– Якщо доступний SSH-доступ, можна запустити додаток через процес-менеджер, наприклад, PM2:

bash
Copy
npm install -g pm2
pm2 start server.js --name "esl-club-app"
pm2 save
Це дозволить вашому додатку працювати у фоновому режимі і автоматично перезапускатися при збоях.

Налаштування проксі-сервера (за необхідності):
– Якщо хостинг використовує Nginx або інший веб-сервер для проксіювання запитів, налаштуйте його так, щоб запити з порту 80/443 перенаправлялися на порт, на якому працює ваш Node.js додаток (наприклад, 3000).
– Приклад конфігурації Nginx:

nginx
Copy
server {
    listen 80;
    server_name esl-club.com;
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
– Для SSL використовуйте Let's Encrypt або інший сертифікат, налаштувавши відповідну конфігурацію.

d. Прив'язка домену
DNS записи:
– Переконайтеся, що ваш домен esl-club.com вказує на IP-адресу вашого сервера (налаштуйте A-запис у панелі керування доменом).

SSL та HTTPS:
– Налаштуйте SSL-сертифікат (наприклад, через Let's Encrypt) для вашого домену, щоб використовувати HTTPS.

3. Тестування продакшн-розгортання
Перейдіть за URL:
– Відкрийте браузер і перейдіть за адресою https://esl-club.com або інший, який ви налаштували.
Перевірте функціональність:
– Переконайтеся, що ваш додаток працює, API викликаються коректно, а база даних підключена.
Перевірте логування:
– Перегляньте логи (якщо є доступ через панель або SSH) для впевненості, що немає помилок.
Підсумок
Підготуйте проект локально (файл .env, залежності, тестування).
Завантажте файли на сервер через FTP або cPanel.
Встановіть залежності та налаштуйте змінні оточення.
Запустіть додаток (через PM2 або налаштування панелі) і переконайтеся, що він працює.
Налаштуйте DNS, SSL і проксі-сервер (Nginx) для вашого домену.
Ці кроки допоможуть вам перенести локальний Node.js-проект на віддалений продакшн-сервер із доменом https://esl-club.com на хостингу ukraine.com.ua.
